# DataMaster 前端菜单重组方案

## 现状分析

通过检查前端路由配置，当前系统包含以下主要模块：
- da: 数据资产
- dm: 数据模型  
- dp: 数据规划
- dpp: 数据开发
- ds: 数据服务
- mc: 元数据采集
- meta: 元数据管理
- sys: 系统管理
- att: 数据质量/属性
- dg: 数据治理
- ai: 人工智能
- example: 示例功能

## 重组目标

参考 DolphinScheduler 的简约设计理念，将前端菜单重组为更清晰、逻辑性更强的结构，提升用户体验。

## 提议的菜单结构

1. **首页** (`/index`)
   - 系统概览和快速入口

2. **数据资产管理** (`/da/*`)
   - 资产地图
   - 数据来源管理
   - 数据目录浏览

3. **数据模型管理** (`/dm/*`)
   - 数据标准
   - 业务词典
   - 模型维护

4. **数据规划** (`/dp/*`)
   - 数据目录
   - 数据元管理
   - 规则模型

5. **数据开发** (`/dpp/*`)
   - 开发任务
   - 作业调度
   - 脚本开发

6. **数据服务** (`/ds/*`)
   - API管理
   - 服务发布
   - 接口文档

7. **元数据管理** (`/mc/*`, `/meta/*`)
   - 元数据采集
   - 元数据浏览
   - 血缘分析

8. **数据质量** (`/att/*`)
   - 质量监控
   - 质量规则
   - 质量报告

9. **数据治理** (`/dg/*`)
   - 安全管理
   脱敏规则
   - 权限管理

10. **人工智能** (`/ai/*`)
    - 智能发现
    - 智能推荐
    - 自动治理

11. **系统管理** (`/sys/*`)
    - 用户管理
    - 角色权限
    - 系统设置
    - 日志监控

12. **示例功能** (`/example/*`)
    - 功能演示
    - 开发示例

## 实现建议

由于菜单动态从后端接口 (`/getRouters`) 获取，真正的菜单重组需要后端配合修改菜单定义。前端可以考虑：

1. 在 `src/store/system/permission.js` 的 `generateRoutes` 或相关过滤函数中添加菜单分组逻辑
2. 修改路由前端处理逻辑，实现自定义菜单排序和分组
3. 更新侧边栏组件 (`src/layout/components/Sidebar/index.vue`) 以支持更好的菜单分组显示
