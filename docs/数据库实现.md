## 3. 能力分层详解

### 3.1 通用层方言 (DialectRegistry) — 基础表/字段元数据查询

注册方式：`DialectRegistry` 构造函数中 `EnumMap<DbType, DbDialect>` 手动注册。

| 数据库 | 方言类 | 继承关系 | 能力说明 |
|--------|--------|----------|----------|
| MySQL | `MySqlDialect` | -> AbstractDbDialect | 完整：列、表、主键、分页 |
| MariaDB | `MariaDBDialect` | -> MySqlDialect | 复用 MySQL |
| Oracle 11g | `OracleDialect` | -> AbstractDbDialect | 完整 Oracle 特有 SQL |
| Oracle 12c+ | `Oracle12cDialect` | -> OracleDialect | 12c 分页优化 |
| PostgreSQL | `PostgreDialect` | -> AbstractDbDialect | pg_attribute/pg_type/pg_constraint 查询 |
| SQL Server 2008 | `SQLServer2008Dialect` | -> AbstractDbDialect | jtds 驱动 + 分页 |
| SQL Server 2012+ | `SQLServerDialect` | -> SQLServer2008Dialect | 新版分页 |
| DM8 | `DM8Dialect` | -> AbstractDbDialect | 达梦专用 SQL |
| Kingbase8 | `Kingbase8Dialect` | -> AbstractDbDialect | 金仓专用 SQL |
| Doris | `DorisDialect` | -> AbstractDbDialect | INFORMATION_SCHEMA.COLUMNS |
| ClickHouse | `ClickHouseDialect` | -> MySqlDialect | system.tables/system.columns |
| Hive | `HiveDialect` | -> MySqlDialect | 复用 MySQL 模式查询 |
| Kafka | `KafkaDialect` | -> AbstractDbDialect | 仅模板，无元数据查询 |
| MongoDB | `MongoDBDialect` | -> UnknownDialect | 仅连接验证 (MongoDB Driver)，元数据抛异常 |
| Elasticsearch | `ElasticsearchDialect` | -> UnknownDialect | 仅 HTTP 连接验证，元数据抛异常 |
| OTHER/HDFS/Phoenix/OSCAR/DB2/Redis/RabbitMQ/FTP/OSS | `UnknownDialect` / `FileDialect` | -> AbstractDbDialect | 所有操作抛 "不支持的数据库类型" |

### 3.2 Catalog 层元数据方言 (DatabaseDialectFactory) — 深度元数据采集

注册方式：`DatabaseDialectFactory` 静态代码块 `HashMap<String, DatabaseDialect>`。

| 数据库 | 方言类 | 能力说明 |
|--------|--------|----------|
| MySQL | `MySqlDialect` | **真实实现**：表结构、列注释、自增、索引、分区 |
| Hive | `HiveDialect` | **真实实现**：DESCRIBE FORMATTED、SHOW PARTITIONS、存储大小、InputFormat → 存储引擎映射 |
| DM8 | `DamengDialect` | **真实实现**：达梦专用元数据查询 |
| Oracle | `AbstractDialect` | **占位** — 无实际实现 |
| PostgreSQL | `AbstractDialect` | **占位** — 无实际实现 |
| SQL Server | `AbstractDialect` | **占位** — 无实际实现 |

### 3.3 行数统计支持 (isCountSupported)

定义位置：`AssetsAssetServiceImpl.java:1574`

支持 9 种类型：`MYSQL`, `ORACLE`, `ORACLE_12C`, `SQL_SERVER`, `POSTGRE_SQL`, `DM8`, `KINGBASE8`, `DORIS`, `HIVE`

### 3.4 增量边界查询 (IncrementalBoundaryQuery)

实现类：`RelationalIncrementalBoundaryQuery.java`

支持 14 种类型：`MYSQL`, `MARIADB`, `ORACLE`, `ORACLE_12C`, `POSTGRE_SQL`, `SQL_SERVER2008`, `SQL_SERVER`, `DM8`, `KINGBASE8`, `PHOENIX`, `DORIS`, `CLICK_HOUSE`, `OSCAR`, `DB2`

### 3.5 数据质量探查 (ComponentRegistry)

注册方式：`ComponentRegistry` 构造器 `Map<String, ComponentItem>`。

| 数据库 | 方言类 | 能力说明 |
|--------|--------|----------|
| MySQL | `MySqlQuality` | 正则 REGEXP, LIMIT/OFFSET 分页 |
| Oracle 11g | `OracleQuality` | Oracle 11g rownum 分页 |
| Oracle 12c+ | `Oracle12cQuality` | 完整覆盖：字符校验、数值校验、组合唯一、分页 |
| SQL Server | `SQLServerQuality` | PATINDEX, LEN, OFFSET/FETCH 分页 |
| DM8 | `DM8Quality` | 空类，全走默认 (ANSI SQL) |
| 其他 | `DefaultQuality` | 默认 ANSI SQL 实现 |

注意：质量探查底层通过 JDBC `Statement.executeQuery()` 执行，非 JDBC 数据库（MongoDB/ES/Redis 等）**无法使用**。

### 3.7 自动建表

入口：`AssetsDatasourceServiceImpl.creaDatasourceTeTable()`  
执行链路：数据源连接校验 → `generateCheckTableExistsSQL()` 判断表是否存在 → `generateCreateTableSQL()` 调用方言 `someInternalSqlGenerator()` → 执行 DDL。

| 数据库 | 自动建表状态 | 说明 |
|--------|--------------|------|
| MySQL / MariaDB | ✓ | `MySqlDialect` 生成 InnoDB DDL |
| Oracle / Oracle 12c | ✓ | `OracleDialect` / `Oracle12cDialect` 生成 DDL 与注释 |
| PostgreSQL | ✓ | `PostgreDialect` 生成 DDL 与注释 |
| SQL Server / SQL Server 2008 | ✓ | `SQLServerDialect` / `SQLServer2008Dialect` 生成 DDL |
| DM8 | ✓ | `DM8Dialect` 生成 DDL 与注释 |
| Kingbase8 | ✓ | `Kingbase8Dialect` 生成 DDL 与注释 |
| Doris | ✓ | `DorisDialect` 生成 Doris DDL，支持分桶/副本扩展入口 |
| ClickHouse | ✓ | `ClickHouseDialect` 生成 MergeTree DDL，主键字段作为 `ORDER BY`，无主键时 `ORDER BY tuple()` |
| Hive | ✓ | `HiveDialect` 生成 Hive 表 DDL，保留字段注释和表注释 |
| MongoDB / Elasticsearch / Kafka / Redis / HDFS / FTP / OSS | — | 非 SQL 表结构，不走 JDBC DDL 自动建表 |
